二次封装vue组件的一种比较好的方式

发表于 2020-07-05 | 分类于 vue   |   字数统计: 1.3k(字)   |   阅读时长: 5(分)

问题

业务中多处使用了 antd-vue 的文件上传组件，因此我封装了一个组件。关键的 fileList 属性我做了双向的绑定。

调用处

封装的组件

现在看起来，这样封装稍微有一点绕，外层调用的 fileList 和封装组件的 files 属性双向绑定，封装组件的 files 又和 a-upload fileList 双向绑定。但是它确实是能工作的，当上传文件后，a-upload 的 fileList 改变了，同时触发 change 事件，handleChange 更新了最外层调用处的 fileList，然后 watch 检测到更新，更新封装组件的 files。

在某处调用的文件上传需要用到更多 a-upload 的 prop 属性进行设置，如果在封装组件上自己手动传递这些属性，太麻烦了，于是想到用 v-bind=$props 进行属性传递，这样做可以把在调用处传递的 prop 全部传给 a-upload。这样的话，fileList 属性也没必要在封装组件里显示传给 a-uplaod,因为 $props 里已经包含了。

对于一般的 props 传递，我还可以理解，但是加上了 .sync 修饰符，也可以用 v-bind=$props 的方式传递吗？我想要实现的是 a-upload 的 fileList 透过封装组件直接和调用处进行双向绑定。

我想了一下，之前看过 .sync 的原理是其实是一个语法糖，在这里相当于在调用处添加了 “update:fileList” 的事件监听。我们知道，vue 的 $emit 只能触发父组件的自定义事件，我想要知道 a-upload 触发双向绑定机制时，更新的是调用处的 fileList 还是封装组件的 fileList。

打印结果是

props 为啥是空的？点开以后，发现实际是有 syncVal 值的，右边有这样的说明 value below was evaluated just now

查了一下，stackoverflow 有一个回答是：

通过 console.log 检查对象以异步方式进行。控制台同步接收对对象的引用，但是直到它被展开后才显示对象的属性(在某些情况下，这取决于浏览器以及在日志发生时是否打开了开发工具)。如果在控制台中检查对象之前对其进行了修改，则显示的数据将具有更新后的值。

也就是说，如果想要打印程序运行时的引用类型的值，需要用 JSON.parse(JSON.stringify(obj)) 来处理一下。

也算是了解了一个小知识点。

2020.7.7 更新

发现之前理解的还是不够透彻，v-bind=$props 是把当前组件的 props 传递下去，如果在当前组件的 props 没有定义接受的属性，$props 里是没有的。所以如果想要实现我想要的效果，正确的做法是使用 v-bind=$attrs，并且在组件中定义 inheritAttrs 属性为 false，同时所有希望传递的 props 都不要在 props 中定义，如果想要访问传递的 prop，可以使用$attrs.xxx。
